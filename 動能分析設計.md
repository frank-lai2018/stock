# 動能分析設計（Momentum Analysis Design）

> 建立日期：2026-07-12
> 對象資料庫：`twstock`（PostgreSQL）；資料範圍 1994-01-05 ～ 2026-07-09，1,942 檔有股價。
> 相依：[資料庫設計.md](資料庫設計.md)、[schema.sql](schema.sql)、每日更新 [daily_update.py](daily_update.py)

---

## 0. 目標與範圍

以既有台股資料庫為基礎，建立一套**可重複、可增量更新**的動能分析框架，支援：

- **選股（as-of 快照）** — 找出當下動能最強的股票（Phase 1，本文件重點）。
- **技術動能指標** — RSI / MACD / KD 等（Phase 2）。
- **回測（backtest）** — 每日橫斷面面板、換股、績效驗證（Phase 3）。

---

## 1. 三個基本原則（避免踩坑）

1. **一律用還原股價 `adj_close` 算報酬。**
   使用未還原的 `close` 會在除權息當天被誤判為暴跌，動能全錯。所有報酬、均線、52 週高低、相對強弱都以 `adj_close` 計算。

2. **universe 過濾（母體篩選）。**
   排名前先剔除失真樣本，否則排行榜會被新股 / 雞蛋水餃股 / 停牌股污染：
   - 上市（有交易資料）滿 **≥ 252 個交易日**（約一年），確保長天期報酬可算。
   - 近 **20 日平均成交金額 ≥ 門檻**（預設 2,000 萬元，可調），濾掉流動性過低者。
   - 最新交易日在合理範圍內（近 N 日內有交易，排除已停牌/下市）。

3. **快照與回測分離。**
   - 選股只需「**最新一天**的橫斷面快照」→ 輕、快、SQL 物化視圖即可。
   - 回測需「**每一天**的橫斷面面板」+ 嚴格防前視（基本面用 `available_date`）→ 重，獨立 Phase 3 處理。

---

## 2. 分層架構

```
┌─ Layer 0  原始表（已完成）
│    price_daily / market_index / inst_trades / margin_trading /
│    monthly_revenue / fundamentals_quarterly …
│
├─ Layer 1  動能特徵（SQL 物化視圖，每日更新後 REFRESH）
│    mv_momentum_snapshot  ← 每檔股票「最新一天」的所有動能因子
│
├─ Layer 2  評分 / 排名
│    universe 內每個因子做百分位排名 → 加權合成 momentum_score
│
└─ Layer 3  應用
     · 選股：SELECT … WHERE score 高 + 濾網（MCP 直接查）
     · 技術指標 RSI/MACD：Python 模組
     · 回測：每日面板 + 換股 + 績效
```

### 在哪算？（關鍵決策）

| 指標類型 | 位置 | 原因 |
|----------|------|------|
| 報酬、均線、52 週高低、相對強弱、量能、籌碼淨額 | **SQL 物化視圖** | window function 一次算全市場，676 萬列也扛得住，MCP 可直查 |
| RSI / MACD / KD 等遞迴型指標 | **Python (pandas)** | SQL 寫起來痛苦，pandas 數行搞定 |
| 橫斷面排名、合成評分 | SQL（`percent_rank()`） | 排名 SQL 最快 |

## 3. 動能因子定義（Layer 1）

設 `D` = 最新交易日；報酬皆以 `adj_close` 計算；「N 個月前」採**日曆基準**取「≤ 該日的最後一筆」還原價，較交易日位移穩健。

### 3.1 報酬類（價格動能核心）

| 因子 | 定義 | 說明 |
|------|------|------|
| `ret_1m` | `adj_close[D] / adj_close[D-1m] - 1` | 近一月報酬 |
| `ret_3m` | 近三月報酬 | |
| `ret_6m` | 近六月報酬 | |
| `ret_12m` | 近十二月報酬 | |
| `ret_12_1` | `adj_close[D-1m] / adj_close[D-12m] - 1` | **經典 12-1 動能**：跳過最近一月，避開短期反轉 |
| `ret_6_1` | `adj_close[D-1m] / adj_close[D-6m] - 1` | 6-1 動能（短波段版） |

> **核心因子建議用 `ret_12_1`（或 `ret_6_1`）**，這是學術與實務最穩健的動能定義。

### 3.2 均線類（趨勢確認）

| 因子 | 定義 |
|------|------|
| `ma20 / ma60 / ma120 / ma240` | 近 N 日 `adj_close` 均值（月/季/半年/年線） |
| `above_ma60` | `adj_close[D] > ma60`（站上季線，布林） |
| `ma_bull_align` | `ma20 > ma60 > ma120`（均線多頭排列，布林） |
| `bias_60` | `(adj_close[D] - ma60) / ma60`（季線乖離率，過高警示追高風險） |

### 3.3 位置類

| 因子 | 定義 |
|------|------|
| `high_52w / low_52w` | 近 252 交易日還原價最高 / 最低 |
| `dist_from_high_52w` | `adj_close[D] / high_52w - 1`（距 52 週高點 %，接近 0 = 創新高） |
| `pct_of_52w_range` | `(adj_close[D]-low_52w)/(high_52w-low_52w)`（在 52 週區間的相對位置 0~1） |

### 3.4 相對強弱（vs 大盤）

| 因子 | 定義 |
|------|------|
| `rs_6m` | `(1+個股ret_6m) / (1+TAIEX同期報酬) - 1`（相對大盤超額，> 0 = 強於大盤） |
| `rs_12m` | 同上，十二月期 |

> TAIEX 取自 `market_index` where `index_id='TAIEX'`。

### 3.5 量能類

| 因子 | 定義 |
|------|------|
| `vol20_avg` | 近 20 日成交量均值 |
| `vol_surge` | `volume[D] / vol20_avg`（爆量倍數） |
| `amt20_avg` | 近 20 日成交**金額**均值（供 universe 流動性濾網用） |

### 3.6 籌碼動能

| 因子 | 定義 | 來源 |
|------|------|------|
| `inst_net_20d` | 近 20 日三大法人淨買超合計（股） | `inst_trades` |
| `inst_buy_streak` | 法人連續買超天數 | `inst_trades` |
| `margin_chg_20d` | 近 20 日融資餘額變化 | `margin_trading` |
| `foreign_ratio_chg` | 外資持股比率近月變化 | `shareholding` |

### 3.7 基本面動能（輔助濾網，選配）

| 因子 | 定義 | 來源 |
|------|------|------|
| `rev_yoy_latest` | 最新月營收 YoY | `monthly_revenue` |
| `rev_yoy_accel` | 月營收 YoY 是否連續加速（近 3 月遞增） | `monthly_revenue` |
| `eps_yoy` | 最新季 EPS 年增 | `fundamentals_quarterly` |

## 4. 評分與排名（Layer 2）

### 4.1 母體（universe）

```
universe = 通過 §1.2 三條濾網的所有股票
```

### 4.2 因子標準化

各因子在 universe 內做**百分位排名** `percent_rank()`（0~1），避免極端值主導、免除量綱差異。方向統一為「越大越強」（乖離率、距高點等反向因子取負或反轉）。

### 4.3 合成分數（預設權重，可調）

```
momentum_score =
    0.40 × rank(ret_12_1)        # 價格動能主軸
  + 0.15 × rank(rs_12m)          # 相對大盤強弱
  + 0.15 × rank(inst_net_20d)    # 籌碼動能
  + 0.10 × rank(vol_surge)       # 量能
  + 0.10 × rank(rev_yoy_latest)  # 營收動能
  + 0.10 × ma_bull_align(0/1)    # 趨勢濾網加分
```

> 權重是起點，非定論；Phase 3 回測後再校準。

### 4.4 濾網（選股時疊加，非計分）

- `above_ma60 = true`（站上季線，避免接下跌刀）
- `dist_from_high_52w > -0.25`（距 52 週高不遠，排除深跌反彈假動能）
- `bias_60 < 上限`（避免乖離過大追在高點，可選）

## 5. 物化視圖規格（Layer 1 產出）

`mv_momentum_snapshot`（主鍵 `stock_id`，每檔一列，as-of 最新交易日）欄位：

```
stock_id, name, market, industry,
as_of_date,
close, adj_close,
ret_1m, ret_3m, ret_6m, ret_12m, ret_12_1, ret_6_1,
ma20, ma60, ma120, ma240, above_ma60, ma_bull_align, bias_60,
high_52w, low_52w, dist_from_high_52w, pct_of_52w_range,
rs_6m, rs_12m,
vol20_avg, vol_surge, amt20_avg,
inst_net_20d, inst_buy_streak, margin_chg_20d,
rev_yoy_latest, rev_yoy_accel, eps_yoy,
in_universe (bool)
```

評分（Layer 2）可另建 `mv_momentum_score`（或在查詢時即時 `percent_rank`），輸出 `momentum_score` 與各因子排名。

## 6. 更新策略

```
每日收盤
  → daily_update.py 灌入當日 price_daily / inst_trades / margin_trading …
  → REFRESH MATERIALIZED VIEW CONCURRENTLY mv_momentum_snapshot;
  → （選股即為最新）
```

- 物化視圖建 `UNIQUE INDEX (stock_id)` 以支援 `REFRESH … CONCURRENTLY`（refresh 時不鎖讀）。
- 月營收 / 財報為低頻更新，隨月/季更新即可。

## 7. 落地路線圖

| 階段 | 內容 | 產出 |
|------|------|------|
| **Phase 1** | 純 SQL：`mv_momentum_snapshot` + 評分查詢 | 立即可用 MCP 選股、看排名（免寫 Python） |
| **Phase 2** | Python 技術指標模組（RSI/MACD/KD） | 補 SQL 難算的遞迴指標，可回寫表 |
| **Phase 3** | 回測：每日面板 + 換股 + 績效（防前視） | 驗證因子有效性、校準權重 |

## 8. 待確認 / 可調參數

- universe 門檻：最少交易日（預設 252）、最低日均成交金額（預設 2,000 萬）。
- 核心動能期間：`ret_12_1` vs `ret_6_1`（長波段 vs 短波段）。
- 合成權重（§4.3）。
- 選股濾網組合（§4.4）。
- 產業中性化：是否在「同產業內」做排名，避免整體集中單一強勢產業（進階選項）。
